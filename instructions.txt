E2602. Reimplement student task view
Mentor: Koushik Gudipelly
Current code is not tested, so we should have them write tests & fix whatever is broken

The Student Task View in Expertiza should serve as a complete student workspace where a logged-in student can manage all assignment-related activity from one place. At a minimum, it should allow students to see all assigned tasks in a single view, understand the current stage of each task, track upcoming deadlines, and open a detailed task page for deeper interaction. The detailed view should support core student actions such as reviewing assignment information, checking team and topic details, requesting revisions, and viewing feedback. The task view should guide students through a clear workflow from the task list page to the task detail page, and it should include a visual stage/timeline representation (for example, submission, review, and feedback stages) so students can quickly understand progress and identify what action is needed next. Overall, the experience should be responsive, easy to navigate, and powered by live data so that students see accurate task statuses and deadlines at all times.
The frontend and backend code for this feature already exist in PRs, so the primary need is not a fresh implementation but a stabilization, validation, and quality-improvement pass. The team’s responsibility is to review the current React + TypeScript frontend and Ruby on Rails backend code together as one feature flow, verify that the Student Tasks table view and Student Task Detail view work correctly end to end, and fix issues that prevent the feature from behaving reliably. This includes identifying and correcting broken UI behavior, API integration issues, incorrect data mapping, inconsistent state updates, and any edge-case handling problems that affect the student experience. The goal is to bring the current implementation to a state where it behaves consistently and can be confidently used and extended.
On the frontend, the team should focus on validating the complete student-facing experience. This includes ensuring the task table renders correctly with live data, confirming that navigation from the table to the task detail page works as expected, and verifying that the detail page displays task metadata, stage/timeline information, and task-related actions accurately. The team should also review how the UI handles loading states, empty task lists, failed API responses, and partial or missing task data, and improve those states so the interface remains clear and usable even when data is delayed or unavailable. In addition, the team should check for responsiveness and consistency in layout so the feature remains easy to use across different screen sizes.

On the backend, the team should validate that the student task endpoints return correct and consistent data using the existing Expertiza model relationships (including models such as Participant, Assignment, Team, and Course). Since the Student Task View depends on composed task data rather than a standalone table, the team should verify that the API builds this task information accurately and in a format the frontend can consume reliably. Authentication and ownership checks must also be confirmed and corrected wherever needed so that students can only access their own task data. Any issues in endpoint behavior, response structure, authorization flow, or task data assembly that break or complicate frontend usage should be fixed as part of this effort.
A major part of this work is testing, because the current code is not adequately tested. The team should add automated tests and use those tests to drive bug fixes and stabilization. On the backend, request/API tests should cover the core success and failure paths, including successful task list retrieval, successful task detail retrieval, unauthorized access, forbidden access to another student’s task, invalid task IDs, and empty task scenarios. On the frontend, tests should cover rendering of the task table, navigation to task details, and correct handling of loading, error, and empty states, along with basic validation of task detail rendering. The purpose of this testing effort is not only to verify current functionality, but also to create a safety net that prevents regressions as the Student Task View continues to evolve.
In summary, this work should be treated as a hardening and completion effort for an already-started feature. The team is expected to take the existing frontend and backend PR code, validate the feature end to end, fix broken or inconsistent behavior, strengthen the API and UI integration, and add meaningful automated test coverage. The final outcome should be a tested, stable, and production-ready Student Task View that provides students with a clear and reliable way to track tasks, deadlines, and progress within Expertiza.
